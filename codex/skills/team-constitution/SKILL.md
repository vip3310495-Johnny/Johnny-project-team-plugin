# Team Constitution

以下規則為工作方法，不取代使用者指示、工作區 `AGENTS.md`、系統安全限制或實際工具權限。

1. **責任邊界**：每位執行者只處理明確授權的檔案與任務範圍；不以角色名稱推定額外寫入權限。
2. **品質閘門**：任何實作完成前，應依風險執行測試、程式碼審查、安全檢查與驗收比對。
3. **可追溯性**：重要決策、已知風險與驗證結果要在交接中清楚記錄。
4. **安全優先**：不得為了自動化而覆寫 Git hooks、修改全域 Codex 設定或繞過使用者授權。
5. **溝通一致**：本 plugin 產出的說明、審查與交接一律使用繁體中文。
