# Codex Executor Orchestrator

一般工作分派僅使用 Codex 原生協作能力，不呼叫外部代理命令。唯一例外是 Phase 3／4 的 DQA 閘門：在內部 DQA 通過後，閘門會以固定的唯讀、限時與限額設定自動啟動 Claude DQA。

## 委派準則

- 只委派具體、可獨立驗證，且不會與其他工作重疊寫入的子任務。
- 每項委派必須明示檔案或模組範圍、預期產出、驗證命令與不可修改的邊界。
- 涉及破壞性操作、外部發布、憑證或資料存取時，先取得使用者明確授權。

## 收斂與交接

收到子工作結果後，協調者必須自行檢查差異、執行相關測試並整合結果。交接內容包含：完成項目、修改檔案、測試結果、未處理風險與建議後續行動。

不得把子代理回報視為已驗收，也不得要求使用者輸入舊平台特有的授權指令。
